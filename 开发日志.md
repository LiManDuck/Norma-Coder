# 参考代码

本项目是以python来实现类claude code的终端编程助手的能力，对于claude code 你可以参考/mnt/e/code/claude-code 这个文件夹（也就是上两层目录） 

# 2026年6月14日 

工具侧实现： 
- 新增subagent 机制，即允许使用Agent工具，输入参数为 prompt, session_id （可选），run_background （默认为false) 三个参数

- 实现一个hook系统，工具权限系统，这两个系统都基于系统总线messagebus来完成，具体的实现可以先简易一点，例如工具权限先设置一个权限模式，三个模式，plan , edit, auto ,  同时允许用户在配置文件的permission 配置各个工具的权限； 对于hook系统，也可以先实现的简单一点，就是允许用户在配置文件的hook 字段，开放一些特定的事件，例如 toolexecuteafter，session-begin , 等，允许配置一个command直接执行


## 实现记录

落地文件：

| 文件 | 说明 |
| ---- | ---- |
| `src/norma/permission/permission.py` | 权限模块：`PermissionMode`、`PermissionDecision`、`PermissionConfig`、`PermissionChecker`，三种 mode（plan/edit/auto）+ per-tool 覆盖。 |
| `src/norma/hook/hook.py` | Hook 系统：订阅 messagebus 事件，执行配置的 shell command，支持 `match` / `timeout` / `background`。 |
| `src/norma/prompt/tool/agent_tool/agent_tool.py` | Agent (subagent) 工具：参数 `prompt` / `session_id` / `run_background` / `description`，支持前台与后台两种模式，session 可复用。 |
| `src/norma/tool/__init__.py` | 工具聚合包，对外稳定导出 `ReadTool / Edit / ... / AgentTool / NormaArtifact`。 |
| `src/norma/agent/norma_coder.py` | 接入 messagebus / 权限 / hook / subagent；工具调用前做权限决策，DENY 转为错误 ToolMessage，ASK 走 messagebus 与用户确认。 |
| `src/norma/cli/cli.py` | 加载 `permission` / `hooks` 配置，启动 messagebus，触发 `session-begin` / `session-end` hook 并注入到 NormaCoder。 |
| `doc/design/permission_hook_design.md` | 设计说明文档（含配置示例、决策流程、消息映射、局限）。 |

要点摘录：

- 默认工具分类：只读 = `Read/LS/Glob/Grep/TodoWrite`，写入 = `Edit/Write`，危险 = `Bash/Agent`；用户在 `permission.tools` 中的设置优先级最高。
- 子 agent 通过工厂注入避免循环依赖；默认 `enable_subagent=False` 在子 agent 中关闭，杜绝递归 spawn。
- Hook 的 `match` 通过命令运行时的环境变量做精确匹配（`TOOL_NAME` / `EVENT` / `CONVERSATION_ID` / `USER_INPUT` 等）。
- ASK 流程：`PermissionChecker` 给出 ASK -> `NormaCoder._ask_user` -> `UserInputManager.request_confirmation` -> messagebus -> 用户。若未注入 `UserInputManager`，按拒绝处理（fail-safe）。

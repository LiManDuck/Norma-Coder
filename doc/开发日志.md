# 工作约束

- 每当你完成某个开发任务时，在这里标记为已完成
- ai-work 文件夹作为你的文件夹, 你可以在其中记录任何你觉得对当前代名仓有利的信息 你将长期参与该代码仓的开发需求 合理利用该文件夹有助于你的开发进展和规划
- 每一个开发需求 其都需要基于广泛的分析实现，选择优秀 符合当前当前代码仓架构的实现方式 而不仅仅作为一个通过测试的任务
- 我仍然建议你对于参考claude code 的代码实现方案，并作为文档不断汇总记录到ai-work下 ; 同时将当前代码仓的架构设计 核心写在其中，并在ai-work下维护一个入口的目录文档
- 当你不知道一个模块该如何实现的时候，可以去参考claude code的设计

## P1 稳定化
- 修复 `openai_llm.py` 未 import `Field`（配置 providers 会 NameError）
- 修复 `AssistantMessage.response` 为必填字段导致构造任何 LLM 响应必崩的 bug（改为默认 None）-> 非/流式 LLM 路径此前实际无法运行

## P2 流式基础
- `LLMResponse` 新增 `stream_reasoning` 增量字段
- 重写 `stream_chat`：逐 chunk yield 增量（stream_content/stream_reasoning），末尾 yield 完整响应（含 usage）
- 新增事件 `AgentTextDeltaEvent` / `AgentThinkDeltaEvent`，`MessageType.AGENT_TEXT_DELTA/AGENT_THINK_DELTA`，`AgentMessageAdapter` 映射
- `NormaCoder.run()` 改用流式 `self.llm(llm_request)`：逐增量 yield delta 事件，捕获最终响应驱动 finish_reason 分支
- 旧 REPL 跳过 delta 事件（最终文本由 LLMResponseEvent 显示），真正逐字流式由 TUI 负责
- 冒烟测试通过

## P3 事件解耦
- 新增 `src/norma/agent/runner.py`：`AgentRunner` 在后台任务驱动 `agent.run()` 生成器；事件由 `NormaCoder._publish` 推送 MessageBus，前端订阅渲染；支持 `cancel()` 中断
- `UserInputManager` 新增 `respond_confirmation(request_id, bool)`：前端确认/拒绝后回送 `USER_CONFIRM/USER_REJECT`，解锁 `request_confirmation` 的 future
- TUI/REPL 不再直接消费生成器；SDK 仍可 `async for event in agent.run(query)`

## P4 TUI 前端（textual 8.x）
- 新增 `src/norma/cli/ui/tui/app.py`：`NormaApp(App)` 主应用，默认前端
- 布局：`RichLog`(history) + `Static`(stream 流式区) + `Static`(status 状态栏) + `Input` + `Footer` + `PermissionModal`(ModalScreen)
- 解耦：`MessageBus.subscribe_all` -> 回调 `post_message(BusEventMessage)` 投递回 UI 线程 -> `on_bus_event_message` 按 msg_type 渲染（与总线同事件循环，无需跨线程）
- 流式：`AGENT_TEXT_DELTA/THINK_DELTA` 追加流式区实时刷新，`AGENT_LLM_RESPONSE` 收尾落盘历史区
- 权限：`UI_PROMPT` -> `PermissionModal`（y 允许 / n,esc 拒绝）-> `respond_confirmation`（headless 验证 allow/deny 闭环）
- 命令统一出口：`CommandContext.print/clear_screen` -> `repl.print_output/clear_screen`；重构 `builtin.py` 全部改用 `ctx.print`，REPL/TUI 复用同一套命令
- 按键：`F2` 切换权限模式、`Ctrl+C` 运行中中断/空闲退出、`Ctrl+L` 清屏
- `cli.py`：默认 `app.run_async()` 启动 TUI，`--repl` 兜底走 prompt_toolkit

## P6 对齐增强
- 工具并发分区：`Tool.is_readonly` 元数据（默认 False）；`NormaArtifact.execute_tools` 按只读/写分区——只读工具 `asyncio.gather` 并发、写工具串行，结果按原序返回；Read/Ls/Glob/Grep/TaskList/TaskGet 标记只读。MCP 工具名前缀 `mcp__server__tool` 已具备。
